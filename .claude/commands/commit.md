Commit and push all current changes to the remote repository.

## Rules

1. **No AI signatures** — never include "Co-Authored-By", "Generated by", or any AI attribution in commit messages.
2. **Professional, detailed messages** — write commit messages that a senior engineer would write. Be specific about what changed and why.
3. **Always push** — after a successful commit, push to the remote branch.

## Steps

1. Run `git status` (never use `-uall`) and `git diff --stat` in parallel to understand current changes.
2. If there are no changes to commit, inform the user and stop.
3. Read the actual diffs (`git diff` for unstaged, `git diff --cached` for staged) to understand what changed.
4. Run `git log --oneline -5` to match the repository's commit message style.
5. Stage relevant files by name (prefer explicit file names over `git add -A`). Never stage `.env`, credentials, or secrets.
6. Write a commit message following this format:
   - **Subject line**: imperative mood, under 72 characters, summarizes the change (e.g., "Add card search and label management to MCP server")
   - **Body** (blank line after subject): explain *what* changed and *why*, organized by logical grouping. Use bullet points for multiple changes. Be specific — mention function names, endpoints, models, etc.
   - Never include any AI co-author lines, signatures, or attribution.
7. Commit using a HEREDOC for the message to preserve formatting.
8. Push to the current remote branch.
9. Report the commit hash and summary to the user.

## Commit Message Style

```
<imperative subject line, max 72 chars>

<grouped description of changes with context>

- Category: specific change details
- Category: specific change details
```

Example:
```
Extend Trello MCP server with card management operations

Add eight new tools for comprehensive board management:
- create_list: create new lists on the configured board
- archive_card: soft-delete cards with reversible archive
- delete_card: permanently remove cards

Infrastructure:
- Add doPost/doDelete HTTP helpers to Trello client
- Introduce CommentResponse and SearchResponse models

Testing:
- Add httptest-based tests for all new client methods
- Add MCP handler tests covering edge cases
```
